# Operation: Push to HQ 🕵️

An interactive, illustrated field manual that teaches complete beginners how to use Git and GitHub — without leaking a single secret. You play *Agent Commit*, delivering your code (intel) to GitHub (HQ).

## What it is

A single, self-contained web page styled like a classified spy dossier — manila folder, typewriter type, rubber stamps — that turns the whole Git/GitHub workflow into a mission. Built for people who've never touched a terminal. The goal: make the first push feel like a cakewalk, and make every command stick.

## What it covers

- **The 4 zones** — an interactive map of working area → staging → local repo → remote, with `git init` shown as the thing that sets it all up
- **The core mission (10 beats)** — clearance & SSH keys, creating a repo, `.gitignore` & secret safety, `init` → `status` → `add` → `commit` → `remote` → `push`
- **The daily loop** — the three commands you'll actually use every day
- **Operation II — Parallel Ops** — branching, pushing a branch, Pull Requests, merging, and resolving merge conflicts, with an interactive branch diagram
- **The field toolkit** — `clone`, `log`, `diff`, and everyday undo with `restore`
- **Bonus op** — the `fork → PR` open-source contribution flow
- **Before you go live** — the team-specific questions to ask and the never-do-this rules for shared/production repos
- **Ambushes & Code Red** — the common errors, and what to do if a secret slips out
- **A cheat card** — the entire thing on one copy-paste index card

## Features for absolute beginners

- 🎯 **Tap-to-copy** on every command — nobody types anything wrong
- 📊 **Progress meters** that fill as you complete each beat, with a "Mission Accomplished" stamp at the end
- ✏️ **Hand-drawn cartoons** for every step
- 🖐️ **Interactive diagrams** — tap a command to light up the zones (or branch stages) it affects
- 🔒 **Security baked into the story** — the recurring villain is *The Leak*; you learn to keep credentials out of your commits
